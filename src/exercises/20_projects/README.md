# Module 20: Final Projects

## Purpose

Apply everything learned in real-world projects.

## Project Ideas

### 1. CLI Application ⭐⭐⭐

Build a command-line tool using `clap`.

- Parse arguments
- Read/write files
- Error handling
- Colored output

**Suggested**: Todo list manager, file organizer, grep clone

### 2. REST API ⭐⭐⭐⭐

Build a web server using `axum` or `actix-web`.

- HTTP endpoints
- JSON serialization with `serde`
- Database integration (SQLite with `sqlx`)
- Error handling
- Middleware

**Suggested**: Task tracker API, URL shortener

### 3. Concurrent Data Processor ⭐⭐⭐⭐

Process data in parallel.

- Multi-threading or async
- Channels for communication
- Progress reporting
- Error aggregation

**Suggested**: Log file analyzer, image resizer

### 4. Simple Database ⭐⭐⭐⭐⭐

Implement a basic key-value store.

- File persistence
- CRUD operations
- Basic indexing
- Thread safety

### 5. TCP Chat Server ⭐⭐⭐⭐⭐

Real-time communication.

- Async networking
- Multiple clients
- Message broadcasting
- Graceful shutdown

---

## Exercise Guidelines

For final projects, the AI should:

1. Help design the architecture
2. Guide through implementation step by step
3. Review code for best practices
4. Suggest improvements
5. Help with testing

Each project should demonstrate:

- Proper module organization
- Error handling
- Testing
- Documentation
- Idiomatic Rust patterns
